# Docタイトルの1行省略表示（truncate）

## 仕様

Docのタイトルを表示する一覧ページ・詳細ページの2箇所は、タイトルがどれだけ長くても1行に収めて表示する。収まらない分は省略記号（…）で表示し、周囲のレイアウト（ボタン等）が崩れることはない。

- 一覧ページ（`app/docs/page.tsx`）
  - 各Docへのリンク（`{doc.title}`）は`block truncate`クラスを持つ。リンク要素をブロック化した上で1行に収まらない分を省略記号で表示する。
  - リンクを囲む`<ul>`・`<li>`はflexコンテナにしない（`divide-y ...`のみを持つ通常のブロック要素）。flexアイテムは既定で「自動最小幅（`min-width: auto`）」を持ち、改行できない長い文字列があると自動最小幅がその文字列の実寸まで広がってしまうため、`<li>`をflexアイテムにしないことで、タイトルの長さに関わらずページ全体の横幅が広がらないようにしている。
- 詳細ページ（`app/docs/[id]/page.tsx`）
  - タイトル見出し`<h1>`は`min-w-0 truncate`クラスを持つ。見出しは「Edit」「Delete」ボタンと横並び（`flex items-center justify-between`）のflexアイテムであり、`min-w-0`によって自動最小幅を無効化することで`truncate`が効き、ボタン側が押し出されない。
  - ボタンを囲む`<div>`は`shrink-0`クラスを持ち、タイトルが長い場合でもボタン領域自体は縮まない。
- ルートレイアウト（`app/layout.tsx`）の`<main>`は`min-w-0`クラスを持つ。左のnavmenu（`<aside>`）と横並びのflexアイテムであるため、ページ内のどこかに改行できない長い文字列があっても、アプリ全体のレイアウト（`<body>`のflex行）が横に広がらないようにする防御的な指定。

## 動作確認

- `npm run build`が正常に完了することを確認済み。
- `npm run test:e2e`で[[../../../tests/doc-title-truncate.spec.ts]]が成功し、非常に長いタイトルのDocを作成した際に、一覧・詳細ページの双方でタイトルが1行に省略表示され、詳細ページの「Edit」「Delete」ボタンが画面内に収まって表示されることを確認済み。
